// 04 — concepts

Concepts
fondamentaux

Les notions clés que tout informaticien doit maîtriser.

⚙️

Algorithmes & Structures de données

Un algorithme est une suite d'instructions pour résoudre un problème. Les structures de données (tableaux, listes, arbres, graphes) organisent les données efficacement. La complexité algorithmique (O(n), O(log n)...) mesure l'efficacité d'un algorithme en temps et en mémoire.

Tri Recherche Récursion Graphes Arbres
🔄

POO — Orienté Objet

Paradigme basé sur des objets qui encapsulent données et comportements. Piliers : héritage, polymorphisme, encapsulation, abstraction.

Classes Héritage Design Patterns
🌐

Réseaux & Internet

TCP/IP, HTTP/HTTPS, DNS, protocoles de communication. Comment les données voyagent d'un point A à un point B à travers le monde.

TCP/IP HTTP REST API
🗄️

Bases de données

SQL pour les données relationnelles, NoSQL pour les données non structurées. ACID, transactions, indexation, optimisation de requêtes.

SQL NoSQL ACID
🔒

Cryptographie

Chiffrement symétrique (AES), asymétrique (RSA), hachage (SHA). Fondement de toute la sécurité informatique moderne.

AES RSA Hash
🧵

Concurrence & Parallélisme

Threads, processus, async/await, coroutines. Exécuter plusieurs tâches simultanément pour maximiser les performances.

Threads Async Mutex

// 06 — histoire

Histoire de
l'informatique

Des grandes étapes qui ont façonné le monde numérique tel qu'on le connaît aujourd'hui.

1936
La machine de Turing
Alan Turing définit les bases théoriques de l'informatique moderne avec sa machine universelle, posant les fondements de la calculabilité.
1969
ARPANET — L'ancêtre d'Internet
Le Département de la Défense américain crée ARPANET, le premier réseau de communication décentralisé, précurseur direct d'Internet.
1971
Premier microprocesseur — Intel 4004
Intel lance le premier microprocesseur commercial. La miniaturisation du calcul devient possible, ouvrant l'ère des ordinateurs personnels.
1991
Le World Wide Web
Tim Berners-Lee invente le Web au CERN. HTML, HTTP et les URLs transforment Internet en un espace accessible à tous.
2007
L'iPhone révolutionne le mobile
Apple lance l'iPhone, redéfinissant l'expérience mobile et lançant l'ère des smartphones et des applications mobiles.
2012
Deep Learning — ImageNet
AlexNet remporte ImageNet avec un réseau de neurones profond, marquant le début de la révolution du deep learning et de l'IA moderne.
2022
L'ère des LLMs — ChatGPT
OpenAI lance ChatGPT, rendant les grands modèles de langage accessibles au grand public et déclenchant une révolution de l'IA générative.
// 07 — glossaire

Glossaire
essentiel

Les termes incontournables à connaître quand on parle informatique.

API
Interface de programmation qui permet à deux applications de communiquer entre elles via des requêtes standardisées (REST, GraphQL...).
Cloud
Infrastructure informatique distante accessible via Internet. Permet de stocker, traiter et déployer sans posséder de matériel physique.
DevOps
Culture et pratiques qui unissent développement et opérations pour accélérer et fiabiliser les cycles de livraison logicielle.
Framework
Ensemble d'outils et de conventions qui structurent le développement d'une application (React, Django, Spring...).
Git
Système de contrôle de version distribué qui trace l'historique du code et facilite la collaboration entre développeurs.
Container
Unité logicielle légère qui emballe le code et ses dépendances pour s'exécuter de manière uniforme sur n'importe quel environnement (Docker).
Machine Learning
Sous-domaine de l'IA où les algorithmes apprennent automatiquement à partir des données sans être explicitement programmés.
Open Source
Logiciel dont le code source est public, librement utilisable, modifiable et distribuable par n'importe qui.
Scalabilité
Capacité d'un système à gérer une charge croissante en ajoutant des ressources (horizontalement ou verticalement).
Microservices
Architecture où une application est divisée en petits services indépendants communicant via des APIs, opposé au monolithe.
Agile / Scrum
Méthode de gestion de projet itérative en sprints courts (2 semaines), favorisant l'adaptation et la livraison continue.
CI/CD
Intégration et déploiement continus — automatiser les tests et le déploiement du code à chaque modification pour livrer plus vite.
